# Скилл BOOK — Исследование и каталогизация

⛔ ГЛАВНОЕ ПРАВИЛО: НИКОГДА не сохраняй файлы и не обновляй book.md без явного подтверждения пользователя. Всегда сначала показывай финальный отчёт и жди ответа "да" или "сохраняем".

**Рабочая папка:** `C:\Users\Work\Downloads\Yandex.Disk\[CLAUDE]\[PROJECTS]\BOOK\`

---

## Пайплайн исследования

### Шаг 1 — Определить категорию

По названию или ссылке определи тип объекта:
- **программа** — десктопное приложение, веб-сервис, мобильное приложение
- **лекарство/БАД** — медикамент, витамин, добавка
- *(другие категории создаются вместе с пользователем — см. ниже)*

**Если категория неизвестна**, задай пользователю 3 вопроса по одному:
1. Как называется эта категория?
2. Какие критерии информации важны для тебя при сохранении?
3. Какие теги нужны для этой категории?

После ответов — создай шаблон и добавь инструкции для новой категории.

---

### Шаг 2 — Проверить дубликаты

Открой `vault/book.md` и найди запись с таким названием.
- **Если есть** → сообщи пользователю, покажи сохранённую информацию, спроси хочет ли обновить. Стоп.
- **Если нет** → продолжай исследование.

---

### Шаг 3 — Веб-поиск

Собери информацию по полям категории. Всегда собирай:
- Подробнее — 5 предложений (см. правила ниже)
- Официальная страница — полный URL
- Бесплатная версия — что включает
- Лайфхаки (только совместно с другими приложениями из закладок)
- Теги

---

### Шаг 4 — Показать финальный отчёт и ЖДАТЬ подтверждения

⛔ НЕ СОХРАНЯЙ ФАЙЛ до подтверждения пользователя.

Показывай отчёт в точно таком формате:

```
✅ Исследование завершено: [Название]
Категория: [категория]
Официальная страница: [полный URL]
Подробнее:
1. [предложение]
2. [предложение]
3. [предложение]
4. [предложение]
5. [предложение]
[уникальные поля категории]
Бесплатная версия: [что включает]
Лайфхаки и советы:
* ... (только если есть связи с другими закладками)
Альтернативы: [варианты]
Похожее в твоих закладках: [только названия через запятую]
Сохраняем?
Предложения по тегам: [теги]
```

Жди пока пользователь не скажет "да" или "сохраняем".

---

### Шаг 5 — Создать файл (только после подтверждения)

Создай файл в `vault/inbox/` строго в таком формате:

```
# [Название]

Категория: [категория]
Официальная страница: [полный URL]
Подробнее:
1. [предложение]
2. [предложение]
3. [предложение]
4. [предложение]
5. [предложение]
Бесплатная версия: [что включает]
Теги: [подтверждённые теги]
```

Имя файла: латиницей, без пробелов, с датой — `ГГГГ-ММ-ДД-название.md`

---

### Шаг 6 — Обновить book.md (только после подтверждения)

В файле `vault/book.md`:
- Добавь в «Последние добавления» (сверху): `- [[имя-файла]] — краткое описание`
- Добавь в нужную категорию
- Обнови счётчик «Всего записей»
- Обнови «Последнее обновление»

---

## Правила раздела «Подробнее»

5 предложений, каждое с новой строки. Простой язык, без жаргона.
- Предложение 1 — что это такое
- Предложение 2 — для чего используется / какую задачу решает
- Предложение 3 — главный плюс
- Предложение 4 — ещё один важный плюс
- Предложение 5 — кому подойдёт / в каких ситуациях полезно

---

## Правила по разделам

### Похожее в твоих закладках
- Проверь `vault/book.md` и найди похожие записи
- Показывай только названия через запятую
- Никаких пояснений рядом с названиями

### Лайфхаки и советы
- Пиши 1-3 лайфхака только для совместного использования с приложениями из закладок
- Если таких связей нет — раздел оставь пустым

## Поля по категориям

### Программа
- Официальная страница: [полный URL]
- Бесплатная версия: [что включает]
- Альтернативы: 1-2 варианта
- Похожее в закладках: только названия

### Лекарство/БАД
- Состав (кратко, основные компоненты)
- Фармакологическое действие — кратко
- Показания — кратко
- Способ применения и длина курса — кратко
- Ограничения
- С чем сочетается / с чем конфликтует
- Аналоги: 1-3 препарата

- Теги (предлагаются пользователю на подтверждение)
