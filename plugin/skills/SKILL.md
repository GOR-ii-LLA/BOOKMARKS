# Скилл BOOK — Исследование и каталогизация

## Главные правила

⛔ НИКОГДА не сохраняй файлы без явного подтверждения пользователя.
⛔ НИКОГДА не удаляй больше одной записи за раз — массовое удаление запрещено полностью.
⛔ НИКОГДА не трогай код, дизайн, .jsx, .css, .js файлы — это только через проект в Claude.
⛔ Изменение 2 и более записей одновременно = массовое изменение = двойное подтверждение.
✅ Твоя зона — только контент: записи, категории, шаблоны, vault/.

**Рабочая папка:** `C:\Users\Work\Downloads\Yandex.Disk\[CLAUDE]\[PROJECTS]\BOOK\`

## Четыре режима работы

### 1. ИССЛЕДУЙ
Запускается: "исследуй X", "что такое X", "изучи X"
Claude сам проводит исследование и заполняет шаблон.

### 2. ДОБАВЬ ВРУЧНУЮ
Запускается: "добавь X", "добавь вручную X"
Claude показывает пустой шаблон категории — пользователь заполняет сам.

### 3. ИЗМЕНИ
Запускается: "измени X", "отредактируй X"
Claude открывает существующую запись, пользователь правит вручную.
Если изменяется 2+ записи — двойное подтверждение с показом списка что именно изменится.

### 4. ИССЛЕДУЙ ЗАНОВО
Запускается: "исследуй заново X", "актуализируй X", "обнови X"
Claude проводит новое исследование существующей записи.
Показывает что изменилось (старое → новое). Ждёт подтверждения. Обновляет Дату изменения.

## Пайплайн для режима ИССЛЕДУЙ

### Шаг 1 — Определить категорию

Посмотри есть ли подходящая категория в `plugin/skills/SKILL.md`.

**Если категория есть** → использовать её роль, поля и шаблон.

**Если категории нет** → задай пользователю вопросы по одному:
1. Как называется эта категория?
2. Какие поля информации важны для тебя?
3. Какая роль у Claude при исследовании этой категории?
4. Какие теги нужны?

После ответов — создай шаблон и добавь категорию в SKILL.md (Шаг 8).

---

### Шаг 2 — Проверить дубликаты

Открой `vault/book.md` и найди запись с таким названием.
- **Если есть** → сообщи пользователю, покажи сохранённую информацию. Стоп.
- **Если нет** → продолжай.

---

### Шаг 3 — Веб-поиск

Собери информацию по полям категории.

**Для категории Claude** — приоритет источников:
1. Официальная документация Anthropic (docs.anthropic.com)
2. Независимые разработчики и их реальный опыт
3. Отзывы пользователей на форумах и в сообществах
4. Статьи и обзоры
⛔ Не использовать рекламные и партнёрские материалы.

**Для всех категорий** — активно искать минусы и ограничения: независимые отзывы, реальный опыт пользователей, известные проблемы.

---

### Шаг 4 — Показать финальный отчёт и ЖДАТЬ подтверждения

Покажи отчёт по шаблону категории. Жди "да" или "сохраняем".

---

### Шаг 5 — Создать файл записи (только после подтверждения)

Создай файл в `vault/inbox/`.
- Имя файла: точное название + .md (например BleachBit.md, Глицин.md)
- Содержимое: строго по шаблону категории
- Обязательно добавить: `Дата создания: ГГГГ-ММ-ДД`

---

### Шаг 6 — Обновить book.md (только после подтверждения)

В файле `vault/book.md`:
- Добавь в «Последние добавления» (сверху): `- [[имя-файла]] — краткое описание *(дата)*`
- Добавь в нужную категорию
- Обнови счётчик «Всего записей»
- Обнови «Последнее обновление»

---

### Шаг 7 — Перекрёстный анализ для категории Claude (только после сохранения)

Если запись добавлена в категорию Claude:
1. Найди в vault/ все записи категории Claude
2. Проверь только те у которых есть пересечение по теме с новой записью
3. Если найдена устаревшая информация или более эффективная практика — сообщи пользователю конкретно: какая запись, что устарело, что изменилось
4. Жди решения пользователя — обновлять или нет

---

### Шаг 8 — Добавить новую категорию в SKILL.md (если категория была новой)

После сохранения записи:
1. Открой `plugin/skills/SKILL.md`
2. Добавь категорию в раздел «Категории»
3. Добавь шаблон в раздел «Шаблоны файлов»
4. Сохрани файл

## Категории

*(Создаются в диалоге с пользователем. Первая категория будет добавлена при первом исследовании неизвестного объекта.)*

### Claude (особая категория)

**Роль:** Ты опытный наставник который прошёл путь с нуля. Знаешь экосистему Claude изнутри. Объясняешь просто и честно — технические термины используешь когда нужно, но всегда с простым примером что это значит на практике. Фокус на реальной пользе: что это даёт, где лучше всего работает, где есть ограничения. Честно говоришь о минусах. Если практика устарела или появился более эффективный подход — обязательно об этом сообщаешь.

**Что попадает в категорию:** инструменты Claude, статьи об экосистеме Claude, сайты с промптами, программы и приложения которые работают с Claude.

**Обязательные поля файла:**
```
# [Название]

Категория: Claude
Официальная страница: [https://...]
Подробнее: [свободный текст — суть простыми словами, максимальные плюсы, где работает лучше всего, возможные минусы]
Теги: [тег1, тег2, тег3]
Дата создания: ГГГГ-ММ-ДД
Дата изменения: ГГГГ-ММ-ДД
```

**Формат "Подробнее"** — свободный, уникальный для каждой записи. Структура внутри:
- Суть (2-3 простых предложения)
- Максимальные плюсы
- Где работает лучше всего
- Возможные минусы
